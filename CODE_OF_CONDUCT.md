# Code of Conduct

Short version: be kind, be honest, argue about code and not about people.

- Harassment, personal attacks, and discrimination are not welcome here, in issues,
  PRs, discussions, or anywhere else this project lives.
- Disagreement is fine and useful. Do it with arguments and benchmarks, not insults.
- Assume good faith first. Most "wrong" reports are a missing piece of context.
- Maintainers may edit, lock, or remove content that crosses these lines, and ban
  repeat offenders.

To report a problem privately, use GitHub's report feature or open a security
advisory if it concerns a vulnerability (see SECURITY.md). Reports are read by the
maintainer and handled as quickly as real life allows.

This is a small project run by one person and a lot of coffee. Kindness scales
better than rules, so the rules stay short.
